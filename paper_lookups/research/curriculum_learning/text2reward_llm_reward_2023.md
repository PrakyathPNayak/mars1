# Text2Reward: Reward Shaping with Language Models for Reinforcement Learning

**Authors:** Xie, T., Zhao, S., Wu, C. H., Liu, Y., Luo, Q., Zhong, V., Yang, Y., Yu, T.
**Year:** 2023 | **Venue:** ICLR 2024
**Links:** [arXiv:2309.11489](https://arxiv.org/abs/2309.11489)

---

## Abstract Summary
Text2Reward proposes using large language models (LLMs) to automatically generate dense reward functions from natural language task descriptions. Rather than manually engineering complex reward functions—a notoriously difficult and time-consuming process in RL—the system takes a textual goal specification (e.g., "make the robot walk forward at 1 m/s while keeping its body upright") and produces executable Python reward code. This generated reward function is then used directly in standard RL training loops (PPO, SAC) to train policies.

The system operates iteratively: an initial reward function is generated by the LLM, a policy is trained, the resulting behavior is evaluated, and feedback (either automated metrics or human observations) is fed back to the LLM to refine the reward. This closed-loop process typically converges to high-quality reward functions within 3–5 iterations. The approach is tested on MuJoCo locomotion tasks (Hopper, Walker, Ant, Humanoid) and manipulation tasks (dexterous hand), achieving performance that matches or exceeds expert-designed reward functions.

Critically, the generated reward code is fully interpretable—researchers can inspect, modify, and version-control the reward function, unlike black-box learned reward models. This transparency makes Text2Reward particularly suitable for safety-critical robotics applications where understanding the optimization objective is essential.

## Core Contributions
- Framework for LLM-based automatic generation of dense reward functions from natural language task descriptions
- Iterative reward refinement loop incorporating training feedback and optional human guidance
- Demonstrated performance matching or exceeding expert-designed rewards on MuJoCo locomotion benchmarks
- Interpretable reward code output (Python functions) enabling inspection, modification, and version control
- Generalizable approach tested across locomotion (quadruped, bipedal) and manipulation domains
- Systematic prompt engineering strategy for eliciting physically grounded reward components from LLMs
- Analysis of failure modes and reward hacking when LLM-generated rewards have unintended optima

## Methodology Deep-Dive
The Text2Reward pipeline consists of three main components: a prompt template, an LLM backbone (GPT-4), and a feedback mechanism. The prompt template provides the LLM with the simulation environment's observation space definition (joint names, body names, sensor descriptions), the action space, and the task goal in natural language. The LLM is instructed to output a Python function `compute_reward(obs, action, info) -> float` that computes a scalar reward given the current observation.

The LLM leverages its training corpus knowledge of physics and robotics to decompose the task goal into physically meaningful reward components. For locomotion, typical components include: forward velocity tracking (reward proportional to `|v_x - v_target|`), upright posture penalty (penalize deviation of torso orientation from vertical), energy penalty (penalize squared joint torques), smoothness penalty (penalize large action changes between timesteps), and ground contact pattern rewards (encourage appropriate foot contact sequences). Each component is weighted, and the LLM proposes initial weights based on common practices.

The iterative refinement loop is the key innovation. After training a policy with the generated reward for a fixed number of steps (e.g., 1M), the system evaluates the resulting behavior against the original task specification. Evaluation metrics include task completion rate, energy efficiency, gait quality scores, and stability measures. These metrics, along with optional human observations ("the robot leans too far forward", "the gait is asymmetric"), are formatted into a feedback prompt and sent back to the LLM. The LLM then modifies the reward function—adjusting weights, adding new terms, or restructuring the computation.

A notable technical detail is the code execution sandbox: the generated reward function is type-checked, dimension-checked, and run through a battery of unit tests (e.g., does it return a finite scalar? Is it differentiable?) before being used in training. This catches common LLM code generation errors early. The system also maintains a reward function history, allowing the LLM to reason about what changes improved or degraded performance across iterations.

For MuJoCo environments specifically, the authors provide the LLM with a structured description of the MuJoCo XML model, including body names, joint limits, and actuator properties. This domain-specific context significantly improves the quality of generated rewards compared to generic prompts.

## Key Results & Numbers
- MuJoCo Ant locomotion: Text2Reward achieves 95% of expert-designed reward performance within 3 iterations
- MuJoCo Humanoid walking: matches expert reward at 98% performance after 5 iterations
- Average reward function generation time: <30 seconds per iteration (GPT-4 API call)
- 3–5 iteration convergence for most locomotion tasks
- Human evaluators preferred Text2Reward gaits over expert-reward gaits in 45% of blind comparisons (statistically tied)
- Manipulation tasks (dexterous hand rotation): 85% success rate vs. 90% with expert rewards
- Reward hacking detected in 15% of initial generations, resolved by iteration 2–3 in all cases
- Code interpretability: average reward function is 20–40 lines of Python, fully human-readable

## Relevance to Project A (Mini Cheetah Quadruped RL)
**Rating: High**
Text2Reward can directly accelerate the reward function design process for Mini Cheetah locomotion training. Designing reward functions for quadruped gaits requires balancing many competing objectives (speed, stability, energy efficiency, foot clearance, body height), and the manual tuning process is one of the most time-consuming aspects of the training pipeline. Text2Reward could generate initial reward functions from natural language specifications like "trot forward at 1.5 m/s on flat ground with minimal energy while maintaining body height at 0.3m."

The iterative refinement loop is especially valuable: after an initial PPO training run, observed gait deficiencies (e.g., excessive body roll, asymmetric stepping) can be described in natural language and fed back to improve the reward. The interpretable code output also enables researchers to understand exactly what is being optimized, making it easier to diagnose reward hacking or unintended behaviors. The MuJoCo compatibility is a direct match for the Mini Cheetah simulation environment.

## Relevance to Project B (Cassie Bipedal Hierarchical RL)
**Rating: High**
Text2Reward has high potential for automating reward function design across all four levels of the Cassie hierarchy. Each level has distinct reward requirements: the Planner level needs rewards for long-horizon goal reaching, the Primitives level needs rewards for diverse skill acquisition (DIAYN/DADS exploration bonuses), the Controller level needs precise joint tracking rewards, and the Safety level (LCBF) needs constraint satisfaction rewards. Manually designing and tuning rewards for all four levels is a massive engineering burden.

The iterative refinement capability is particularly valuable for the hierarchical setting, where reward interactions between levels can produce emergent behaviors (both positive and negative) that are difficult to predict. Text2Reward could be used to rapidly prototype and iterate on reward functions at each level, with feedback from end-to-end evaluation guiding the refinement. The interpretability of generated rewards also aids in debugging inter-level reward conflicts. For the Dual Asymmetric-Context Transformer, Text2Reward could generate context-dependent reward modulations based on natural language descriptions of terrain or task conditions.

## What to Borrow / Implement
- Integrate Text2Reward into the Mini Cheetah training pipeline for rapid reward prototyping: specify desired gaits in natural language, train, evaluate, iterate
- Use the iterative refinement loop to systematically improve Cassie hierarchy rewards level-by-level, starting from the Controller and working upward
- Adopt the code sandbox and unit testing approach for validating any manually designed reward functions
- Leverage the MuJoCo observation space description format as a template for providing environment context to the LLM
- Build a reward function library by generating Text2Reward outputs for multiple locomotion objectives (walking, trotting, bounding, turning) and curating the best versions

## Limitations & Open Questions
- Dependent on LLM quality: GPT-4 produces significantly better rewards than GPT-3.5; performance may vary with different LLMs
- Reward hacking remains a risk—LLMs can generate rewards with unintended optima that produce high reward but undesirable behavior
- The iterative feedback loop requires human judgment for nuanced behavioral evaluation, limiting full automation
- Not yet tested on real hardware; generated rewards may not account for sim-to-real gaps in reward component sensitivities
